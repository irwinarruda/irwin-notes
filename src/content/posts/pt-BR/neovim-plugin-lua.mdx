## Por que Lua?

Vimscript foi importante por muito tempo, mas Lua traz um fluxo moderno de script para o Neovim.

- **Performance** - LuaJIT e mais rapido para tarefas pesadas
- **Sintaxe familiar** - quem usa JavaScript ou Python pega o jeito rapido
- **API de primeira classe** - `vim.api`, `vim.fn` e `vim.opt` cobrem quase tudo
- **Ecossistema alinhado** - plugins como telescope e treesitter sao Lua-first

## Estrutura do projeto

A estrutura de pastas e previsivel e escala bem:

```text title="meu-plugin.nvim"
meu-plugin.nvim/
|- lua/
|  |- meu-plugin/
|  |  |- init.lua
|  |  |- config.lua
|  |  '- core.lua
|- plugin/
|  '- meu-plugin.lua
|- doc/
|  '- meu-plugin.txt
'- README.md
```

`lua/` guarda seus modulos. `plugin/` e carregado automaticamente pelo Neovim e e um bom lugar para comandos e inicializacao.

## Escrevendo o core

Este exemplo monitora variacao de numero de linhas ao salvar:

```lua title="lua/meu-plugin/core.lua" {11-19}
local M = {}

local state = {}

function M.on_buf_write(bufnr)
  local lines = vim.api.nvim_buf_line_count(bufnr)
  local prev = state[bufnr]

  if prev and lines ~= prev then
    local delta = lines - prev
    local sign = delta > 0 and "+" or ""
    vim.notify(
      string.format("%s%d lines (%d total)", sign, delta, lines),
      vim.log.levels.INFO
    )
  end

  state[bufnr] = lines
end

function M.setup()
  vim.api.nvim_create_autocmd("BufWritePost", {
    callback = function(ev)
      M.on_buf_write(ev.buf)
    end,
  })
end

return M
```

## Registrando comandos

Comandos deixam o plugin mais facil de descobrir:

```lua title="plugin/meu-plugin.lua"
if vim.g.loaded_meu_plugin then
  return
end

vim.g.loaded_meu_plugin = true

vim.api.nvim_create_user_command("MeuPluginAtivar", function()
  require("meu-plugin.core").setup()
  vim.notify("meu-plugin ativado", vim.log.levels.INFO)
end, { desc = "Ativa o rastreamento de linhas do meu-plugin" })
```
